# Forecasting *Aedes aegypti* Abundance in Southern California

Code repository for the manuscript:

> Sehi, G. et al. (in review). "From Ecology to Intervention: Modeling Invasive
> *Aedes aegypti* Dynamics in Southern California." *Ecological Applications.*

---

## Overview

This repository contains reproducible R code for two complementary forecasting analyses of *Aedes aegypti* trap surveillance data (2017–2023) from the San Gabriel Valley Mosquito and Vector Control District, Southern California.

**Analysis 1 — Static forecasting** (`04_static_models.R`):
Models trained on 2017–2022 data and evaluated on held-out 2023 trap counts.
Predicts same-week mosquito abundance (rate per trap-night).

**Analysis 2 — Rolling 2-week-ahead forecasting** (`05_rolling_forecast.R`):
Models re-trained each week using all data available up to that point,
predicting *Aedes aegypti* abundance 2 weeks forward. This is the primary
analysis reported in the manuscript.

Three models are compared against two baselines (persistence and seasonal
climatology) using MAE, RMSE, R², and forecast skill score:

| Model | Description |
|-------|-------------|
| **GAM** | Generalized Additive Model with negative binomial family, spatial smooth, seasonality, environmental covariates, and AR lags |
| **RF** | Random Forest on log-rate with UTM coordinates as spatial features |
| **RFsp** | Random Forest with kernel PCA spatial features (Nyström projection) replacing raw coordinates |

## Data Availability

**Trap surveillance data** are provided by the San Gabriel Valley Mosquito and
Vector Control District and are available upon reasonable request to the
corresponding author. Due to data sharing agreements, raw trap data are not
included in this repository.

**Environmental covariates** (NDWI, 14-day lagged precipitation, 14-day lagged
temperature, digital elevation model, built environment index) were derived from
publicly available remote sensing sources described in the manuscript Methods.

**To use this code with your own data**, your input data frame (`SBD_17_23_xy`)
must contain the following columns (see `R/01_data_prep.R` for details):

```
site_code, collection_date, disease_week, Total_Aedes,
longitude, latitude,
NDWI14_scale, Psum14_scale, Tlag14_scale,
DEM_scale, Built_scale
```

**Shapefiles** for the study-area boundary and California counties are not
included due to file size. See `inst/boundaries/README.md` for sources and
download instructions.

## How to Run

1. Clone this repository and open `aedes_forecasting.Rproj` in RStudio.
2. Restore the package environment: `renv::restore()`
3. Place raw data in `data/raw/` and shapefiles in `inst/boundaries/`
   (see `inst/boundaries/README.md`).
4. Run scripts in order:

```r
source("R/00_config.R")            # load packages, set paths, load helpers
source("R/01_data_prep.R")         # → data/processed/site_week.rds
source("R/02_covariates.R")        # → data/processed/site_week_full.rds
source("R/03_eda_figures.R")       # → figures/Figure1_operational_context.pdf
source("R/04_static_models.R")     # → figures/Figure2_*.pdf, output/metrics_static.csv
source("R/05_rolling_forecast.R")  # → data/processed/roll_results.rds
source("R/06_spatial_maps.R")      # → figures/Figure_spatial_predictions.pdf
source("R/07_figures_metrics.R")   # → figures/Figure3–6.pdf, output/metrics_rolling.csv
```

Total runtime is approximately 30–60 minutes on a standard laptop,
with the rolling forecast loop (`05`) being the most time-intensive step.
